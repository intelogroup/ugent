Proposed Design: Hybrid Medical RAG

Optimized for Accuracy, Context, and Scale

1. Chunking Strategy (Hierarchical)

We won't just split by character count. We will respect the Markdown structure:

  • Small Chunks (256-512 tokens): For specific facts/mnemonics.
  • Medium Chunks (1024 tokens): For whole subsections (e.g., "Nucleotides").
  • Overlap: 15% overlap to maintain context between chunks.

2. Metadata Enrichment (The "Breadcrumb" Secret)

To prevent the AI from getting lost, every chunk will be prepended with its location:

[Book: First Aid] > [Section: Biochemistry] > [Topic: DNA Methylation]
---
"DNA is methylated in imprinting. Methylation within gene promoter..."

3. Image Integration (Text-Linked)

We store the image_id and filename as metadata fields in the vector index. When a text chunk near an image is retrieved, the UI can immediately display the corresponding image from extracted_images/.

4. Pinecone Configuration

Index Settings

  • Model: text-embedding-3-large
  • Dimensions: 3072
  • Metric: Cosine Similarity

Scaling

  • Namespace: "first-aid-2023", "pathoma-2021"
  • Filters: Use Pinecone metadata filtering for book-specific search.
