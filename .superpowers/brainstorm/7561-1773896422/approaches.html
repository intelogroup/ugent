Architecture Approaches

Choosing how the chatbot interacts with Pinecone and your images

A

Simple API Route (Linear RAG)

Frontend sends prompt → API Route queries Pinecone → API Route calls OpenAI → Response with images returned.

  • Pros: Very simple to implement, predictable.
  • Cons: No streaming by default, harder to handle complex multi-turn logic.
B

AI SDK Streaming (Recommended)

Uses streamText and useChat. LLM streams the response. We inject context from Pinecone into the prompt.

  • Pros: Modern "ChatGPT" feel, built-in streaming, easy to add tool-calling later.
  • Cons: Requires slightly more boilerplate for context injection.
C

Agentic Tool Calling

LLM decides when to "search_textbooks". It explicitly asks for Pinecone data only when needed.

  • Pros: Most flexible, can handle "Tell me about X" and "Find an image of Y" differently.
  • Cons: Slower (extra round trip), potentially higher token usage.

Image Strategy

For all options, we'll serve the Git LFS images from a dedicated /public/images folder or an external S3/Vercel Blob bucket so the chatbot can display them via Markdown or custom components.
