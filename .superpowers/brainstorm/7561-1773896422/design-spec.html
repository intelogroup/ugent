Key Components

  • Retriever: A utility function that mirrors your embed_books.py logic to vectorize user queries and fetch Top-K chunks from Pinecone.
  • Image Server: We'll map the image_ids from Pinecone metadata to the files in /public/extracted_images.
  • Context Injector: A prompt template that wraps retrieved textbook snippets into the system message.

Why this works

  • Fast, responsive UI with real-time streaming.
  • Low latency retrieval from Pinecone.
  • Reuses your existing embedding model configuration.

Considerations

  • Need to ensure Next.js image optimization handles 2k+ local images efficiently.
  • Requires environment variables for Pinecone/OpenAI on Vercel.
